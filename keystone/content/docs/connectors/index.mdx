## Common Setup

All connectors require authentication credentials. You can provide them via:

1. **Environment variables** (recommended for CI/CD)
2. **Configuration file** (`~/.keystone/config.toml`)

### Environment Variables

```bash
export VERCEL_TOKEN="your-token"
export NETLIFY_AUTH_TOKEN="your-token"
export RENDER_API_KEY="your-api-key"
# etc...
```

### Configuration File

```toml
[connector_auth]
vercel_token = "your-token"
netlify_auth_token = "your-token"
render_api_key = "your-api-key"
```

## Usage Pattern

All connectors follow the same usage pattern:

```bash
# Rotate a secret
keystone rotate SECRET_NAME --env ENV --service PROVIDER

# With redeploy
keystone rotate SECRET_NAME --env ENV --service PROVIDER --redeploy

# Dry run first
keystone rotate SECRET_NAME --env ENV --service PROVIDER --dry-run
```
